# axisml-lite

The **no-Kubernetes, single-host** deployment form of AxisML. It runs the control plane and infrastructure on one Linux host via Docker Compose, and drives user training jobs, workspaces and online services as Docker containers through the Docker Engine API.

Lite shares the Platform API, the `MLRun` / `MLService` / `MLTrafficPolicy` workload contract, the PostgreSQL schema and the main user flows with the **Standard** (full Kubernetes) form. It is fixed to a single tenant (`default`), a single host and a single control-plane replica.

## What's inside

A single Go binary, **`axisml-core`**, folds the three System modules (Cluster Manager, Compute Service, Artifact Hub) and an in-process **Standalone Compute Runtime** onto one router. The runtime maps the AxisML workload contract to Docker containers, volumes, networks and a Traefik file-provider config ‚Äî it is the only process holding the Docker socket. `axisml-platform` runs as a separate process and reaches `axisml-core` over HTTP, exactly as in the Standard form.

## Configuration

`axisml-core` runs under Docker Compose, where **the environment is the only configuration source** ‚Äî this binary reads no config file. Every key is supplied as its `AXISML_` variable, following the repo-wide [configuration conventions](../docs/configuration.md):

- **Key ‚áÑ env var:** `<section>.<key>` ‚áÑ `AXISML_<SECTION>_<KEY>` (uppercased; dots ‚Üí underscores).
- **Precedence:** built-in default < `AXISML_<KEY>` env var < `AXISML_<KEY>_FILE` secret file (secret keys only; highest priority).
- **Fail-fast:** a missing required key (e.g. `database.host`) or a validation error leaves the process not ready.

This reference is generated by `go run ./axisml-core/cmd/config-doc-gen` over the `Config` struct ‚Äî regenerate it when the struct changes. axisml-core is intentionally **excluded** from the repo-level `docs/configuration.md` per-service manual because it is env-only and configured via its Compose environment block; this table is its home.

| Key | Environment variable | Default | Secret | Description |
|---|---|---|---|---|
| `database.host` | `AXISML_DATABASE_HOST` | `localhost` | ‚Äî | PostgreSQL host |
| `database.port` | `AXISML_DATABASE_PORT` | `5432` | ‚Äî | PostgreSQL port |
| `database.name` | `AXISML_DATABASE_NAME` | `axisml` | ‚Äî | Database name |
| `database.user` | `AXISML_DATABASE_USER` | `axisml` | ‚Äî | Database user |
| `database.password` | `AXISML_DATABASE_PASSWORD`<br>`AXISML_DATABASE_PASSWORD_FILE` | ‚Äî | yes | Database password |
| `database.sslmode` | `AXISML_DATABASE_SSLMODE` | `disable` | ‚Äî | libpq sslmode: disable \| require \| verify-full |
| `log.level` | `AXISML_LOG_LEVEL` | `info` | ‚Äî | Log level: debug \| info \| warn \| error |
| `log.format` | `AXISML_LOG_FORMAT` | `json` | ‚Äî | Log format: json \| console |
| `oci.endpoint` | `AXISML_OCI_ENDPOINT` | `http://zot:5000` | ‚Äî | OCI registry endpoint (full URL; scheme derived from it) |
| `oci.admin_user` | `AXISML_OCI_ADMIN_USER` | `admin` | ‚Äî | OCI registry admin username |
| `oci.admin_password` | `AXISML_OCI_ADMIN_PASSWORD`<br>`AXISML_OCI_ADMIN_PASSWORD_FILE` | ‚Äî | yes | OCI registry admin password |

Operational parameters ‚Äî bind address, filesystem paths, Docker network, background cadences ‚Äî are **not** environment-configurable. They are fixed constants for the Lite binary (`core.DefaultSettings`), overridable only when `axisml-core` is embedded as a library via `core.WithSettings`.

## Build / run

```sh
make help                  # list targets
make build                 # build the axisml-core binary
make test                  # unit tests (no Docker)
make image                 # build the axisml-core image (build context = repo root)
make doc-gen / doc-test    # regenerate / verify the axisml-core OpenAPI spec

make lite-up               # bring up the stack: db + axisml-core on :8090 + platform UI on :8080
make lite-down             # tear down (CLEAN=1 also removes data volumes)
make lite-delete           # purge the stack + all axisml-managed workload containers & volumes
make e2e-test              # run the centralized e2e suite against the running stack
```

`make lite-up` brings up the full control plane ‚Äî PostgreSQL, `axisml-core` and the `axisml-platform` API + UI (browse to `http://localhost:8080`). The remaining services are profile-gated; pass `PROFILES="storage gateway"` to add the artifact stores (zot ¬∑ RustFS) and the Traefik gateway:

```sh
make lite-up PROFILES="storage gateway"
```
